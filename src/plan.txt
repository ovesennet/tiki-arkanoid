Core rules of Arkanoid
You control the Vaus paddle at the bottom of the screen and keep the ball in play while clearing the stage’s breakable bricks. If the ball falls below the paddle, you lose a life. The C64 version has 5 lives to start. The game has 32 regular stages, then a boss fight as stage 33. 
A stage is cleared when all breakable bricks are gone. Indestructible bricks do not count toward stage completion. The paddle only moves horizontally, and the bounce angle depends on where the ball hits the paddle. 
Brick and power-up behavior
Arkanoid uses three practical brick categories:


Normal colored bricks: destroyed in one hit.


Silver bricks: require multiple hits.


Gold / indestructible border bricks: cannot be destroyed and are not required to finish the level. 


The power capsules in the C64 version are:


S = Slow


C = Catch


E = Enlarge


D = Disruption / 3 balls


L = Laser


B = Break to next level


P = Extra life 


Also, the C64 version awards an extra life every 20,000 points. 
How levels progress
Level progression is mainly:


Start stage with ball attached to paddle.


Launch ball.


Destroy all required bricks.


Advance to next stage.


Difficulty rises through layout complexity, tighter channels, tougher silver bricks, more awkward rebounds, and generally faster pace. Silver bricks need more hits as the game advances. 


The B capsule is important: it opens an exit on the right side, and sending the ball through it skips directly to the next stage and gives a 10,000 point bonus. 
What matters if you want it close to C64
The parts to imitate are:


Precise paddle feel


Predictable but controllable bounce angles


Ball speed ramping


Stage-specific brick layouts


Power-up timing and drop rate


Very clean collision handling


Fast redraw without flicker


The C64 game also supports paddles/mouse and feels much better with analog-like control, which is part of why joystick versions often feel harsher.  For TIKI-100, even if you use digital input, you should try to simulate a responsive, low-latency paddle.

Collision design
4. Ball physics
Do not implement free-angle physics. Arkanoid feels better with a controlled set of rebound angles.
Use fixed-point movement, for example:


x, y in 8.8 fixed point


dx, dy as signed 8.8 vectors


Recommended bounce model
Divide paddle into 5 zones:


far left → shallow left


left center → medium left


center → steep upward


right center → medium right


far right → shallow right


That is closer to arcade logic than real physics, and easier to tune.
Important rule
When the ball hits:


walls: invert dx


ceiling: invert dy


brick: determine whether horizontal or vertical side hit dominates, then invert appropriate axis


paddle: assign new dx/dy from paddle zone table


This is much more stable than trying to “simulate” real reflection.

Rendering strategy
5. Use a static playfield plus dynamic objects
Since Arkanoid has a mostly static background:


draw full level once


erase/redraw only changed bricks


redraw paddle, balls, capsules, enemies each frame


use dirty rectangles or XOR-free redraw logic


This is a very good fit for an 8-bit machine.
Suggested layers:


background / border


brick field


HUD


moving objects



Level format
6. Build a compact stage data format
Store stage data as compressed rows.
Example:
typedef struct{    uint8_t m_brickType;    uint8_t m_capsuleType;} BrickCell;
But for storage, compress it more tightly:


4 bits brick type


4 bits capsule type


Then unpack into RAM when loading stage.
If memory gets tight, keep:


packed stage ROM data


unpacked active stage RAM grid



Power-up implementation order
7. Implement capsules in this order
Build them in the order that helps gameplay testing most:


E enlarge


S slow


P extra life


C catch


L laser


D disruption


B break


Why this order:


first three are simplest


catch changes launch behavior


laser introduces shots


disruption introduces multiple balls


break requires stage exit logic


Notes


Only one main paddle mode should usually be active at once, except extra life which is instantaneous.


Losing the ball should clear most temporary paddle powers. StrategyWiki notes that power-up effects last until another power-up is collected or the current ball is lost. 



Level progression and difficulty
8. Difficulty model for your version
Even if you cannot match the original exactly at first, use this progression:
Early stages


simple walls


fewer silver bricks


slower starting ball


Mid stages


more narrow channels


more gold blockers


more silver bricks


more awkward returns


Late stages


dense layouts


more forced side rebounds


faster ball acceleration


higher enemy interference


You should also increase silver-brick toughness with stage number, as the original progression does.

Practical simplifications for first version

Tuning priorities
12. The three things to tune hardest
If you want it to feel right, spend most time on:


paddle movement latency


ball rebound table


collision stability around brick corners


Those three will determine whether it feels “real” or homemade.
